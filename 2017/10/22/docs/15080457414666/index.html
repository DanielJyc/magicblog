AB过滤掉：否

  • 操作员维度：
    • 消费总金额：SUM(Q)，即消费金额求和
    • 对客编去重后
      • 累计可分期金额：SUM(P 累计可分期金额)；==或者SUM(AD)，分期金额求和？==
  • 客编维度：
    • 消费总和：SUM(Q)，即消费金额求和
    • 分期金额总和：SUM(AD)，即分期金额求和

AB过滤掉：否、空白

  • 客编维度：
    • 成功户数：COUNT(E)，即客编的数量；==需要去重吧？ 减5什么意思？==
    • 分期总期次(汇总的平均数)：AVG(AC)，即分期期数的平均数
  • 客编+分期金额阈值[0,5000] [5000,10000] [10000,20000] [20000,50000] [50000,100000] [100000, 正无穷]
    • 户数：COUNT(E)，即6个区间的客户数
  • 操作员维度：
    • 分期金额总和：SUM(AD)，即分期金额求和
    • 分期手续费总和：SUM(AE)，即分期手续费求和
    • 对客编去重后
      • 分期期数的平均数：AVG(AC)
    • 他下面的3、6、9、12、18、24期的数
  • 分期期数
    • 客户数总和：COUNT(E)
