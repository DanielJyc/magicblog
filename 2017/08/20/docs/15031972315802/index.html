网格智能运营

网格0817

本周主要是联调和发布阶段。负责重点区域区域和复合人群的开发联调。

  • LBS相关的负责结构树组装，对外屏蔽；同时，二叉树切为多叉树模式。
    • LBS数据集查询：在da进行包装。之前，开发的时候忽略了热力图模式，周四进行了兼容支持。
    • 创建重点区域或LBS人群：最初开发的时候使用了二叉结构树模式，本周发现：当使用100个POI的时候，树的深度是99，组装出来的sql将会嵌套99层，导致性能问题。修复为使用DS的多叉树能力，并在DA将树封装为对象。
  • nomo的一些需求整理：对nomo的一些功能进行了整理，下周同步给一风，再统一给nomo提需求。其中有个问题，nomo不支持不规则区域和POI条件的复用，商量后，在DA层进行了网格拆分。最后，在语法验证的时候，发现nomo将两类地理位置信息做了and关系，不符合产品需求。跟pd商量后，在页面将两类分开建立。
  • da优化整理：本期功能完成的同时，还有一些地方需要进一步优化改进，对其做了整理
  • 数据一致性：均挚本次提供了pgsql的runner，同时在两个数据集上面进行了验证。但是，由于pgsql精度更高一些，将会导致这两个数据集跟其他数据，口径对不上。因此，本次仍然全部采用ads。后续，考虑将一个独立模块进行引流。

其他

  • mif查询报错：周五上午，mif调用ds由十几个报错，和若道一起排查后，发现是参数中心配置没有获取到。找崇幻确认后，发现参数中心在mrchdt和其自身系统，均无报错，并且数据也加载到了。虽然报错数量不多，但是该机器为灰度机器，调用量总量很少，从该机器的比例来看，并不低。重启该机器后，无报错出现；后面，找pe确认，该机器也并没有问题。该问题，还需要后续的继续观察。
